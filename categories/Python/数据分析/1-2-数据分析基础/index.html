数据挖掘的流程 (Knowledge Discovery in Database,KDD)

任务
  • 分类
    • 通过训练集得到一个分类模型，然后用这个模型可对其他数据进行分类
    • 训练集：作为机器分类的训练样本(已经分类好的数据)
    • 测试集：用来评测分类模型(已经分类好的数据)
  • 聚类
    • 将数据自动聚类成几个类别(没有先验知识)
  • 预测
    • 通过当前和历史数据来预测未来趋势
  • 关联分析
    • 发现数据中的规则
      流程
  • 数据预处理
    • 数据清洗：去除重复数据，去噪声以及填充缺失值
    • 数据集成：多个数据源的数据存放在统一的数据存储中
    • 数据变换：将数据转换成适合数据挖掘的形式
  • 数据后处理：将模型预测的结果进一步处理后，再导出？

用户画像：标签化就是数据的抽象能力

用户画像的准则
  • 统一化：同一用户的唯一标识
    • 标识物：用户名、注册手机号、联系人手机号、邮箱、设备号、CookieID等
    • 作用：更好的跟踪和分析用户的特征
  • 标签化：用户消费行为分析
    1. 用户标签：用户的基础属性。(性别、年龄、地域、收入、学历、职业等)
    2. 消费标签：用户的消费习惯。(消费习惯、购买意向、是否对促销敏感等)
    3. 行为标签：用户使用App的习惯。(通过分析用户的行为：时间段、频次、时长、访问路径)
    4. 内容分析：用户感兴趣的内容。(通过用户浏览内容时，对停留时间长、浏览次数等行为，分析用户感兴趣的内容：金融、娱乐、教育、体育、时尚、科技)
  • 业务化：指导用户画像(应用,带来价值：获取新的，留住在的，分析走的)
    1. 获客：如何进行拉新，通过更精准的营销获取客户。
    2. 粘客：个性化推荐，搜索排序，场景运营等。
    3. 留客：流失率预测，分析关键节点降低流失率。
数据流划分用户画像建模的过程
  • 数据层：获取用户消费行为里的标签。(事实标签,数据客观的记录)
  • 算法层：通过用户的行为进行建模。(模型标签,建立用户画像)
  • 业务层：获客，粘客，留客的手段。(预测标签)
实操：美团外卖的用户画像？
  1. 统一化：同一用户的唯一标识
    　
    美团与大众点评已经合并，选择手机号，两个数据库可以互通
  2. 标签化：按照“用户消费行为分析”的准则设计
    1. 用户标签：性别、年龄?、地域(家乡?、居住地?、收货地址)、收入?、职业?、婚姻?、宝宝信息?、何种渠道注册
    2. 消费标签：口味、消费均价、团购等级、预定使用等级?、排队使用等级?、外卖等级
    3. 行为标签：点外卖时间段、点外卖频次、点餐时长、访问路径?
    4. 内容标签：基于用户平时浏览的内容进行统计，包括餐饮口味、优惠敏感度等
  3. 业务
    通过数据挖掘，分析其中关联()
    预测用户行为
    产生业务价值:
    • 获客: 吸引有潜在需求的用户,并刺激其转化；(寻找优势的宣传渠道,个性化的宣传手段)
    • 粘客: 提升用户的单价和消费频次；(用户购买后的个性化推荐,针对优质用户进行优质高价商品的推荐、以及重复购买,比如通过红包、优惠等方式激励对优惠敏感的人群,提升购买频次)
    • 留客: 预测用户是否可能会从平台上流失 (用户流失可能会包括多种情况,比如用户体验、竞争对手、需求变化等,通过预测用户的流失率可以大幅降低用户留存的运营成本。)
